# ML-сервис для предсказания одобрения ипотеки

## Общая информация о проекте

Данный проект представляет собой консультирующий веб-сервис, который предсказывает вероятность одобрения или отказа в выдаче ипотеки на основе предоставленных данных клиента. Сервис реализован в виде веб-приложения с использованием FastAPI для backend-части и предоставляет API для взаимодействия с моделью машинного обучения. Frontend-часть представляет собой адаптивный веб-интерфейс для удобного взаимодействия пользователя с сервисом.

Проект выполнен в рамках лабораторной работы по дисциплине "Программирование на Python". Разработка велась командой из трёх человек, каждый из которых отвечал за свою область.

## Запуск ML-сервиса

### Установка и запуск backend-сервера

Клонируйте репозиторий и перейдите в директорию проекта:

```bash
git clone https://github.com/Anne4ka-rom/Cheremsha.git
cd Cheremsha/fastapi-app
```

Установите необходимые зависимости:

```bash
pip install -r requirements.txt
```

Запустите backend-сервер:

```bash
uvicorn main:app --reload
```

Сервер будет доступен по адресу: `http://127.0.0.1:8000`

### Запуск frontend-интерфейса

Откройте файл `index.html` в любом браузере. Для работы всех функций также потребуются файлы `predict-csv.html` и `upload-model.html`, которые должны находиться в той же директории.

## ML-компонент

### Общая информация

ML-модуль отвечает за разработку и обучение модели машинного обучения для предсказания вероятности одобрения ипотеки. Модель является основой всего сервиса и интегрируется с backend-частью через сериализованный пайплайн.

### Выполненные задачи

**Анализ и подготовка данных**

Проведён первичный анализ датасета. Исходные данные содержали 13 признаков и 45000 записей без пропусков. Выявлены типы данных, проверен баланс классов целевой переменной (22% одобрений, 78% отказов), обнаружены выбросы в числовых признаках.

Выполнена очистка данных. Пропуски в числовых признаках заполнены медианными значениями, в категориальных - модальными на случай подключения другого датасета с пропусками. Выбросы обработаны методом межквартильного размаха - этот метод выбран из-за его устойчивости к аномалиям и отсутствия требования нормального распределения данных. Удалены аномальные значения в признаках возраста, дохода, суммы кредита и кредитного рейтинга. После очистки в датасете осталось 38577 записей.

Проведена визуализация ключевых закономерностей. Построен график, показывающий распределение кредитного рейтинга и доли дохода на кредит в зависимости от статуса одобрения. Визуализация подтвердила логичность данных: клиенты с высоким кредитным рейтингом и низкой долговой нагрузкой чаще получают одобрение.

**Преобразование признаков**

Категориальные признаки (person_gender, person_education, person_home_ownership, loan_intent, previous_loan_defaults_on_file) закодированы с использованием One-Hot Encoding с параметром drop='first' для избежания мультиколлинеарности. Числовые признаки масштабированы с помощью StandardScaler для приведения их к единому диапазону. Общее количество признаков после кодирования составило 22.

Для снижения размерности и улучшения обобщающей способности модели проведён отбор признаков с использованием метода SelectKBest на основе F-статистики. Отобрано 15 наиболее информативных признаков, которые обеспечивают наилучшее качество предсказания.

**Обучение и выбор модели**

Обучены две модели машинного обучения:

- Random Forest с количеством деревьев 150 и максимальной глубиной 12. Выбор этих гиперпараметров обусловлен необходимостью баланса между качеством обучения и скоростью предсказания.
- XGBoost с количеством деревьев 150, глубиной 6 и скоростью обучения 0.1. Градиентный бустинг выбран как один из лучших алгоритмов для табличных данных.

Оценка качества проводилась по метрике ROC-AUC на тестовой выборке, составляющей 20 процентов от всех данных (7716 записей). Для проверки стабильности моделей использована пятикратная кросс-валидация на обучающей выборке.

Результаты кросс-валидации показали, что XGBoost демонстрирует среднюю ROC-AUC 0.9759 +/- 0.0009, а Random Forest - 0.9710 +/- 0.0005. Лучшей признана модель XGBoost, показавшая ROC-AUC на уровне 0.9773 на тестовой выборке. Random Forest показал сопоставимый результат (0.9720), но уступил по точности. Значение AUC = 0.9773 означает, что модель с вероятностью 97.73% правильно ранжирует случайную пару "одобрение - отказ".

**Визуализация результатов**

Построена ROC-кривая лучшей модели с нанесёнными порогами классификации и областью сравнения со случайной моделью. Диагональная линия на графике соответствует случайному угадыванию с AUC равным 0.5. Чем выше ROC-кривая модели находится над этой диагональю, тем качественнее предсказание. Полученное значение AUC = 0.9773 говорит об отличном качестве модели и её пригодности для использования в реальном сервисе.

**Сохранение модели**

Создан полный пайплайн, объединяющий все этапы обработки данных (масштабирование, кодирование, отбор признаков) и классификатор XGBoost. Это позволяет применять модель без повторной реализации предобработки на стороне backend. Пайплайн сохранён в формате .pkl с использованием библиотеки joblib.

Дополнительно сохранён файл features.json, содержащий список отобранных признаков, название лучшей модели и достигнутое значение ROC-AUC для удобной интеграции с backend-частью.

### Результаты работы ML-части

В результате выполнения ML-компонента получены следующие артефакты:

- Файл model.pkl содержит полностью готовый к использованию пайплайн, объединяющий все этапы предобработки и классификатор XGBoost
- Файл features.json содержит метаинформацию о модели: список из 15 отобранных признаков, название лучшей модели (XGBoost) и значение ROC-AUC (0.9773)
- График best_model_roc.png демонстрирует ROC-кривую лучшей модели с визуализацией порогов классификации и сравнением со случайной моделью
- График eda_analysis.png визуализирует распределение кредитного рейтинга и доли дохода на кредит в зависимости от статуса одобрения

Итоговая метрика ROC-AUC на тестовой выборке составляет 0.9773, что соответствует 97.73% правильных ранжирований и подтверждается результатами пятикратной кросс-валидации (0.9759 +/- 0.0009)

## Backend-компонент

### Общая информация

Backend-модуль отвечает за создание API-сервиса, интеграцию ML-модели и обеспечение взаимодействия между фронтендом и моделью машинного обучения. Сервис реализован на FastAPI и предоставляет три основных эндпоинта для работы с моделью.

### Выполненные задачи

**Разработка API-эндпоинтов**

Эндпоинт /upload-model принимает POST-запрос с файлом модели в формате .pkl, сохраняет его на сервере и загружает в память для дальнейшего использования. При успешной загрузке возвращается статус успеха, при ошибке - соответствующий код ошибки.

Эндпоинт /predict принимает POST-запрос с JSON-объектом, содержащим признаки клиента. Эндпоинт валидирует входные данные с помощью Pydantic-схем, передаёт их в загруженную ML-модель и возвращает предсказание в виде вероятности одобрения от 0.0 до 1.0.

Эндпоинт /predict-from-csv принимает POST-запрос с CSV-файлом. Эндпоинт обрабатывает файл, делает предсказания для каждой строки, добавляет колонку с предсказанным статусом. Если в исходном файле присутствовала целевая переменная loan_status, дополнительно рассчитывается и возвращается метрика ROC-AUC в заголовке ответа.

**Интеграция ML-модели**

Реализован класс ModelController для работы с ML-моделью, который загружает model.pkl при вызове /upload-model. Класс предоставляет методы для предсказания одного объекта и обработки CSV-файлов. Модель не требует дополнительной предобработки на стороне backend - пайплайн выполняет все необходимые преобразования автоматически.

**Обработка ошибок и валидация**

Реализована обработка всех возможных ошибок: отсутствие загруженной модели возвращает статус 405 с сообщением об ошибке, невалидные входные данные возвращают статус 422, ошибки при загрузке или чтении файлов возвращают статус 500 с соответствующим сообщением.

**Тестирование**

Написаны тесты с использованием FastAPI TestClient и Pytest. Файл конфигурации тестов - pytest.ini. Тесты покрывают следующие сценарии: проверка эндпоинта /health, успешная загрузка модели, загрузка неверного формата файла, получение предсказания для валидных данных, обработка ошибки при отсутствии модели, эндпоинт /predict-from-csv с валидным и невалидным CSV-файлом. Все тесты успешно проходят.

**Логирование**

Настроено логирование с помощью перенаправления логов uvicorn из терминала в файл. Файл конфигурации логирования - log.ini. В лог записываются дата и время каждого запроса, метод и путь эндпоинта, статус ответа, сообщения об ошибках. Это позволяет отслеживать работу сервиса и диагностировать возникающие проблемы в production-среде.

### Результаты работы Backend-части

В результате выполнения backend-компонента получены следующие результаты:

- Разработано полностью функционирующее API с тремя эндпоинтами, соответствующими техническому заданию
- Написан класс ModelController для интеграции ML-модели, который автоматически выполняет всю предобработку данных без необходимости дублирования кода на стороне backend
- Созданы Pydantic-схемы для валидации всех входящих запросов с подробными сообщениями об ошибках
- Написано 7 тестов, успешно проходящих все сценарии использования API
- Настроено логирование всех запросов и ошибок в файл logfile.log

## Frontend-компонент

### Общая информация

Frontend-модуль отвечает за создание пользовательского интерфейса для взаимодействия с сервисом. Интерфейс позволяет пользователю вводить данные о себе, отправлять запрос к API и получать предсказание о вероятности одобрения ипотеки в наглядной форме.

### Выполненные задачи

**Структура интерфейса**

Разработано три HTML-страницы:

- index.html - основная страница для ввода данных одного клиента и получения предсказания
- predict-csv.html - страница для загрузки CSV-файла с пакетной обработкой данных
- upload-model.html - страница для загрузки обученной модели в формате .pkl

Все страницы имеют единый тёмный минималистичный дизайн и навигационную панель для переключения между режимами работы.

**Форма для одиночного предсказания**

Создана HTML-форма, содержащая все необходимые поля для сбора 13 признаков клиента. Реализована валидация всех полей: убрана возможность ввода отрицательных значений, установлено возрастное ограничение от 18 до 100 лет. При ошибках валидации пользователь видит подсветку некорректных полей и сообщение над кнопкой отправки.

**Функционал загрузки модели**

Реализована страница для загрузки модели с кастомным файловым инпутом, отображением имени выбранного файла и кнопкой очистки. После успешной загрузки модели отображается уведомление об успехе.

**Функционал обработки CSV-файлов**

Реализована страница для загрузки CSV-файлов с валидацией расширения и структуры. После обработки файла отображается метрика ROC-AUC (если целевая переменная присутствовала в исходном файле), количество обработанных записей и кнопка для скачивания результата.

**Отправка запросов к API**

Реализована функция отправки POST-запросов на соответствующие эндпоинты. Запросы отправляются асинхронно с помощью fetch API. Во время ожидания ответа кнопка отправки блокируется и отображается индикатор загрузки.

**Отображение результатов**

Реализовано отображение результата предсказания: вероятность одобрения преобразуется в проценты, при значении 50 процентов и выше выводится "ОДОБРЕНО" зелёным цветом, при значении ниже 50 процентов - "ОТКАЗАНО" красным цветом. Добавлена визуальная индикация в виде прогресс-бара, заполняемого в соответствии с вероятностью.

**Обработка ошибок**

Реализована обработка всех возможных ошибок: при статусе 405 выводится сообщение "Модель не загружена", при ошибках сети или CORS выводится понятное сообщение пользователю, при ошибках валидации выполняется подсветка пустых и некорректных полей.

### Результаты работы Frontend-части

В результате выполнения frontend-компонента получены следующие результаты:

- Созданы три HTML-файла, содержащие полностью готовый пользовательский интерфейс для всех режимов работы сервиса
- Реализована форма с валидацией всех 13 полей, соответствующая требованиям технического задания
- Написана JavaScript-логика для отправки асинхронных запросов к API и обработки ответов
- Реализована визуализация результата предсказания с прогресс-баром и цветовой индикацией
- Добавлена обработка всех возможных ошибок с понятными пользователю сообщениями на русском языке
- Создан адаптивный тёмный дизайн, корректно отображающийся на мобильных устройствах и десктопах

**Что проверяется в CI:**

- Корректность форматирования кода
