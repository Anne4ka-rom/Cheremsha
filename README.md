# Cheremsha
Командный проект "ML-сервис для предсказания ипотеки"

## Фронтенд 

Для реализации был использован Deepseek.

**Пример промпта для `predict.html`:**

Ты - senior frontend-разработчик. Сгенерируй файл predict.html для интерфейса сервиса предсказания одобрения ипотеки.

HTML-форма должна собирать исходные признаки:

    "person_age",                    // число
    "person_gender",                 // категория: Male/Female
    "person_education",              // категория: Bachelor/High School/Master/Doctorate
    "person_income",                 // число
    "person_emp_exp",                // число
    "person_home_ownership",         // категория: RENT/OWN/MORTGAGE/OTHER
    "loan_amnt",                     // число
    "loan_intent",                   // категория: EDUCATION/HOMEIMPROVEMENT/MEDICAL/PERSONAL/VENTURE
    "loan_int_rate",                 // число
    "loan_percent_income",           // число
    "cb_person_cred_hist_length",    // число
    "credit_score",                  // число
    "previous_loan_defaults_on_file" // категория: Yes/No
    
Изначально поля должны быть пустыми для заполнения. Убери возможность ставить отрицательные значения, возрастное ограничение: от 18 до 100. Кнопка: «Получить предсказание». Блок вывода результата отображается после отправки формы. Темный, минималистичный, адаптивный дизайн с прямоугольными полями.

Форма должна отправлять POST-запрос на эндпоинт /predict с телом { "info": { ... } }. Ответ: { "info": {...}, "loan_status": float } // вероятность одобрения 0.0–1.0. Отображение: преобразовать loan_status в проценты, показать "ОДОБРЕНО" при ≥50%, "ОТКАЗАНО" при <50%, добавить визуальную индикацию (прогресс-бар/цвет).

Обработка ошибок:
- 405: "Модель не загружена"
- CORS: понятное сообщение
- Валидация: подсветка пустых/некорректных полей

**Побочные промпты для создания `upload-model.html` и `predict-from-csv.html`:**

После получения рабочего predict.html дальнейшие страницы создавались по аналогии с требованиями: сохранить единый стиль (тёмная тема, навигация), адаптировать логику под соответствующий эндпоинт (/predict-from-csv или /upload-model), добавить специфические элементы (кастомный файловый инпут, отображение ROC-AUC, возможность скачать результат).
