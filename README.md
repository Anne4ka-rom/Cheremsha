Командный проект "ML-сервис для предсказания ипотеки"

# ML

Я отвечала за разработку и обучение модели машинного обучения для предсказания вероятности одобрения ипотеки. Модель является основой всего сервиса и интегрируется с backend-частью через сериализованный пайплайн.

---
## Выполненные задачи
В рамках работы над ML-компонентом были выполнены следующие этапы:

**Анализ и подготовка данных**

- Проведён первичный анализ датасета. Выявлены типы данных, проверен баланс классов целевой переменной, обнаружены выбросы и обнаружено отсутствие пропусков. Анализ показал, что датасет содержит `13 признаков` и требует предобработки перед обучением.

- Выполнена очистка данных. Пропуски в числовых признаках заполнены медианными значениями, в категориальных - модальными (на случай, если будет подключён другой датасет с пропусками). Выбросы обработаны с помощью метода межквартильного размаха, что позволило удалить аномальные значения в признаках возраста, дохода, суммы кредита и кредитного рейтинга.

- Проведена визуализация ключевых закономерностей. Построен график, показывающий распределение кредитного рейтинга и доли дохода на кредит в зависимости от статуса одобрения. Визуализация подтвердила логичность данных: клиенты с высоким кредитным рейтингом и низкой долговой нагрузкой чаще получают одобрение.

**Преобразование признаков**

- Категориальные признаки закодированы с использованием `One-Hot Encoding` с параметром `drop='first'` для избежания мультиколлинеарности. Числовые признаки масштабированы с помощью `StandardScaler`. Общее количество признаков после кодирования составило `25`.

- Для снижения размерности и улучшения обобщающей способности модели проведён отбор признаков с использованием метода `SelectKBest` на основе `F-статистики`. Отобрано `15` наиболее информативных признаков, которые обеспечивают наилучшее качество предсказания.

**Обучение и выбор модели**

Обучены две модели машинного обучения:

- *Random Forest* с количеством деревьев `150` и максимальной глубиной `12`

- *XGBoost* с количеством деревьев `150`, глубиной `6` и скоростью обучения `0.1`

Оценка качества проводилась по метрике `ROC-AUC` на тестовой выборке, составляющей `20%` от всех данных. Для проверки стабильности моделей использована пятикратная кросс-валидация.

Лучшей признана модель `XGBoost`, показавшая `ROC-AUC` на уровне `0.9773`. Random Forest показал сопоставимый результат, но уступил по точности.

**Визуализация результатов**

Построена ROC-кривая лучшей модели с нанесёнными порогами классификации и областью сравнения со случайной моделью. Диагональная линия на графике соответствует случайному угадыванию с `AUC = 0.5`. Чем выше ROC-кривая модели находится над этой диагональю, тем качественнее предсказание. Полученное значение `AUC = 0.9773` говорит об отличном качестве модели.

**Сохранение модели**

Создан полный пайплайн, объединяющий все этапы обработки данных и классификатор. Это позволяет применять модель без повторной реализации предобработки на стороне backend. Пайплайн сохранён в формате `.pkl` с использованием библиотеки `joblib`.

Дополнительно сохранён файл `features.json`, содержащий список отобранных признаков, название лучшей модели и достигнутое значение `ROC-AUC`.

**Результаты работы**

В результате выполнения ML-части получены следующие артефакты:

- Файл `model.pkl` содержит полностью готовый к использованию пайплайн

- Файл `features.json` содержит метаинформацию о модели, необходимую для корректной интеграции

- График `roc_curves.png` демонстрирует качество модели и может быть использован в отчётных материалах

- График `eda_analysis.png` визуализирует ключевые закономерности в данных, подтверждающие адекватность выбранных признаков

- Итоговая метрика ROC-AUC на тестовой выборке составляет `0.9773`, что свидетельствует об отличном качестве предсказаний и пригодности модели для использования в сервисе

**Использованные инструменты**
Для реализации ML-компонента использовались следующие библиотеки:

- `pandas` и `numpy` для обработки и анализа данных

- `scikit-learn` для предобработки, отбора признаков и реализации случайного леса

- `xgboost` для градиентного бустинга

- `matplotlib` для визуализации

- `joblib` для сериализации модели
